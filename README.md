Each company has its own expectations regarding the level of knowledge required for each role. These guidelines are based on my experience and on the current standards used in the company where I work.

## Intern-Level Coding Interview — Candidate Guide

### Who this is for

Candidates currently enrolled in a Computer Science or related undergraduate/graduate program. 

We assume that candidates have taken introductory courses in:

- logic and algorithms
- programming fundamentals
- basic mathematics

Candidates are not expected to have taken advanced courses such as:

- databases
- compilers
- operating systems
- advanced object-oriented programming

The interview focuses primarily on general problem-solving ability and foundational coding skills.

### What to expect

The interview combines conceptual discussion and practical coding exercises.

Candidates may use common programming languages such as:

- Java
- Python
- C
- C#

Candidates should choose the language they are most comfortable with.

The interview is divided into four main areas:

#### 1. Logic & Problem Solving

Candidates may receive short puzzles or simplified real-world problems.

Before writing code, candidates are expected to explain their approach and reasoning.

We are evaluating how the candidate:

- understands the problem
- breaks the problem into smaller steps
- explains the reasoning behind the proposed solution

#### 2. Programming Foundations

Questions about core programming concepts, such as:

- variables
- control flow (if / loops)
- functions or methods
- basic data structures (arrays / lists)

The goal is to verify that the candidate understands the fundamental building blocks of programming.


#### 3. Implementation

Candidates will solve small programming exercises using straightforward algorithms and standard language features.

Evaluation focuses on:

- correctness
- readability
- handling of simple edge cases

Advanced frameworks or libraries are not required.


#### 4. Basic System Design (Feature Thinking)

A lightweight discussion about implementing a simple feature, such as a to-do list.

We may discuss:

- requirements
- basic components
- simple data flow
- basic trade-offs

The objective is not architecture depth, but to evaluate structured thinking

## Junior Level Code Interview

A Junior developer is expected to have most of the core knowledge typically covered in a Computer Science undergraduate program.

This includes the following areas:

### 1. Programming Language Fundamentals

Candidates should be comfortable with the fundamental constructs of at least one programming language.

This includes:

- functions / methods
- conditional statements
- loop structures
- basic object-oriented programming concepts

Depending on the role, familiarity with other programming paradigms may also be helpful, such as:

- functional programming concepts
- structs or simple data containers


### 2. Problem solving skills

Problem-solving ability is one of the most important skills for a developer, and it is often underestimated.

Many candidates focus heavily on learning syntax or frameworks. However, programming is fundamentally about **solving problems using a computer**. Languages and frameworks are simply tools used to translate a problem into instructions that a computer can execute.

A strong developer must be able to:

- understand the problem
- think of possible solutions
- translate the solution into code

If you don't know this, knowing framework is useless.

To practice problem-solving skills, platforms such as the following are recommended:

- [Vjudge](https://vjudge.net/)
- [HackerRank](https://www.hackerrank.com/)
- [Beecrowd](https://www.beecrowd.com.br/judge/pt/login?origem=1)


### 3. Concepts of relational database

Most software systems interact with databases, so basic knowledge of relational databases is important. 

Candidates should understand:

- [Basic SQL syntax](https://www.w3schools.com/sql/)
- [Database Management System (DBMS)](https://www.techopedia.com/definition/24361/database-management-systems-dbms)  


They should also be able to write simple queries, such as:

- INSERT, UPDATE, and DELETE
- SELECT statements
- SELECT with JOIN
- sorting and filtering data


### 4. Version Control Systems 

Create code with a team means dealing with code synchronization and keeping track of different versions of the project. It would be great if there were tools to help with that, right?

Good news: there are!

Tools like [Git](https://git-scm.com/) or Mercurial exist exactly for this purpose. They help developers organize and manage their code by doing things such as:

- Providing an easy way to version your code, allowing you to access different points in the development history;
- Setting tags to mark special versions of the code (usually releases);
- Showing in a clear way the changes someone is proposing, so the team can review whether those changes make sense;
- And many other useful things that make collaborative development much easier.

### 5. Reading and Understanding Errors

Modern compilers and runtimes provide detailed error messages.

Developers should learn how to read and interpret these messages to identify the source of a problem.

Understanding error messages is an important skill for diagnosing issues during development.


### 6.Familiarity with Common Programming Errors 

Candidates should recognize and understand common mistakes that occur during development, such as:

- array index out of bounds
- null pointer exceptions
- off-by-one errors
- incorrect loop conditions

## Pleno Level Code Interview

A Mid-Level developer is expected to have solid experience with all the concepts described for the Junior level.

At this stage, the developer should not struggle with basic concepts and should be able to learn new tools and technologies quickly.

In addition to the Junior-level expectations, a Mid-Level developer should also demonstrate:

- stronger debugging and investigation skills
- better code organization and readability
- the ability to reason about trade-offs in implementation
- autonomy in solving moderately complex problems
- the ability to work productively with existing codebases

More advanced topics such as system design, architecture decisions, and performance considerations may also be explored depending on the role.
